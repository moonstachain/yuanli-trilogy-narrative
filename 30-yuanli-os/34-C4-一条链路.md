# C4 · 一条链路

> 器官定位：**手脚**。白话：让系统能行动。上一步接 [C3 一张地图](33-C3-一张地图.md)，终点接回 [C1 一纸文脉](31-C1-一纸文脉.md) 与 [总飞轮](../00-canon/03-总飞轮.md)——这里是莫比乌斯环合拢的地方。

## 1. 旧认知隔离

"链路"最容易被降格理解成"把事做完的流程"。五条隔离：

- **不是待办事项。** 待办清单记录"要做什么"，链路规定"这类事永远怎么做"。清单划掉一项就消失，链路跑完一次会更强。
- **不是临时任务。** 临时任务的生命周期是一次；链路的判据是可重复——同类任务第二次来，能不能不重新发明一遍做法。
- **不是一次性交付。** 交付完成、款项两清、文件归档，看起来圆满，实际上是系统层面的流产：这次交付里长出的方法、踩过的坑、校准好的标准，一样都没有留下。
- **不是自动化脚本堆砌。** 脚本自动化"手"，链路装配"手 + 验收 + 回写"。只有前者，你得到的是一堆跑得飞快但从不长进的机械臂。
- **不是没有验收标准的流程。** 写了步骤没写"做到什么程度算过"，等于把质检永久外包给你本人的在场——那流程交给谁都不放心，交给 AI 更不放心。

一句话隔离：**做完了事什么都没留下的，叫劳动；做完了事系统变强了的，才叫链路。**

## 2. 小切口

原力体系创始人复盘过自己最漂亮的一次交付，结论让他坐不住。

那个项目各方面都堪称满分：客户极满意、续约、转介绍。但三个月后同类项目再来，他发现团队几乎在从零重做——上次的流程散在几个人的临时分工里，上次的验收标准只存在于他当时的口头把关里，上次踩过的坑被换了个姿势重新踩了一遍。最讽刺的是：那次交付中他和 AI 协作打磨出的一套极好用的工作方法，因为没有写下来，连他自己都只记得"当时好像很顺"。

满分交付，零分沉淀。他算了一笔账：如果每次行动的经验留存率是零，那么做一百个项目和做一个项目，对系统而言没有区别——公司规模在长，公司能力曲线是一条水平线。**问题不出在执行力，出在每条流程的最后都缺同一节：回写。** 事情做完的那一刻，恰恰是系统本该开始学习的一刻，而他的系统在那一刻下班了。

## 3. 大纵深

**历史维度：现代工业的分水岭不是机器，是"可交接的流程"。** 手工作坊时代，品质住在师傅的手里，师傅走了品质就走了；流水线革命真正的发明不是传送带，而是把"怎么做、做到什么标准"从人身上剥离出来写进工序——从此产能第一次可以复制、可以移交、可以在创始工匠不在场时保持水准。今天创始人要把任务交给 AI 和团队，面对的是同一道门槛：**你交出去的到底是"一个任务"，还是"一套带输入、输出、验收、禁区的作业"？** 前者交出去是风险，后者交出去才是杠杆。

**时代维度：AI 执行力的溢出，让"链路化程度"成为新的分界线。** 这一代 Agent 已经不缺手脚，缺的是被说清楚的活儿。同样一件事，说不清验收标准的人只能得到平均水平的敷衍，而一条写清输入、输出、文件、验收、禁止事项、回写位置的链路，能让 AI 稳定产出高标准结果——并且每跑一轮，把结果沉回大脑。人与人的差距，正在从"谁更会干"变成"谁的干法可以被机器执行"。

**理论维度：组织学习的经典分野是单环与双环。** 单环学习修正行动（这次哪里做错了），双环学习修正行动背后的规则（我们的标准、地图、乃至"我们是谁"是否该更新）。链路的前六链完成执行，第七链回写把单环升级为双环——任务的产出不止是交付物，还有对文脉、大脑、地图的修订提案；第八链再进一步，把反复被验证的套路蒸馏成 Skill，让"学到的东西"本身成为可调用的构件。**这正是这个系统敢自称生命体的原因：它有代谢。**

## 4. 新植入

### 一句话定义与核心公式

> **一条链路，是把战略地图中的判断，转化为可重复执行、可交给人或AI、可验收、可回写、可自动化的行动流程。**

```text
一条链路 = 判断 → 执行（可重复 × 可交接 × 可验收）→ 回写 → 蒸馏
第一性问题：如何把战略判断变成可复用、可执行、可自动化的行动？
```

关键边界：链路不是"把事情做完"，而是让每一次行动都留下——输入、输出、验收标准、角色分工、工具调用、回写位置、下一次如何更强。

| # | 判断标准 |
|---|---|
| 1 | 是否能重复执行？ |
| 2 | 是否能交给别人或AI？ |
| 3 | 是否有明确输入输出？ |
| 4 | 是否有验收标准？ |
| 5 | 是否有回写位置？ |
| 6 | 是否能沉淀成 SOP、Agent 或 Skill？ |

**输出**：一条链路 / SOP / Agent workflow / Codex-ready task / 自动化流程 / Evolution Note / Skill蒸馏卡。
**回写位置**：OS链路、codex/、project/、templates/、skills、原力母体升级记录。
**完成标志**：它能让同类任务下次更快、更准、更少依赖人肉重来。

### 回写链：飞轮在此合拢

C4.7 是整个原力OS 的闭环点。任务结束，产出写回四个器官与母体：文脉更懂你（口径校准）、大脑更厚（案例与样例入库）、地图更准（阈值与因果被验证或修正）、链路更快（SOP 迭代）、母体升级（"我是谁"被市场结果重新照亮）。四步之环接回 C1，OS 之环接回母体——**母体 → 原力 → 市场 → 生意 → OS → 母体升级**，详见 [总飞轮](../00-canon/03-总飞轮.md)。做会自我繁殖的事（[母公理ʸx](../00-canon/01-母公理ʸx.md)），在行动层的全部含义就是这一节。

## 5. 高弧光

有人问：把一切写成 SOP、交给 Agent，人不就成了自己流程里的零件？

看清回写链的人会给出相反的答案。没有链路的世界才真正把人当零件用——你被钉在每一个执行环节，因为流程离了你就散，那不是被需要，是被消耗。链路把可重复的部分交给系统之后，留给你的是零件干不了的事：立法（改文脉）、审美（校样例）、拍板（越阈值的判断）、以及在回写时刻决定"这次经历应该让我们变成什么"。而最动人的是那条飞轮的终点：**每一次行动最终回写的是母体——你出发时相信的那件事，被一次次真实世界的往返，磨得越来越清晰、越来越有力。** OS 不是让你消失在系统里，是让你第一次有一个系统，配得上你最初的那道光。

## 6. 可复用金句

- 做完了事什么都没留下的，叫劳动；做完了事系统变强了的，才叫链路。
- 满分交付，零分沉淀——公司在长大，能力曲线是水平线。
- 交出去"一个任务"是风险，交出去"一套带验收的作业"才是杠杆。
- AI 不缺手脚，缺的是被说清楚的活儿。
- 事情做完的那一刻，是系统开始学习的一刻——别让你的系统在那一刻下班。
